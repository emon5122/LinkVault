# Play Console — exact answers for LinkVault

Every Google Play requirement, with the precise answer to enter. Answers reflect what LinkVault
actually does: **no accounts, no data collection, no ads, no tracking, fully offline.** Uploadable
items are handled by the pipeline; the items below marked ✍️ are console attestations only you can
certify — but there's no decision to make, just enter the value shown.

> Legend: 🤖 automated by the pipeline · 🎨 graphic asset (generated/added once) · ✍️ you enter/confirm

### Permission footprint (verified via `expo prebuild`)

`app.json → expo.android.blockedPermissions` strips permissions that dependencies inject but the app
doesn't use, so the store shows a clean, honest list:

| Permission | Result |
| ---------- | ------ |
| `INTERNET` | ✅ kept — fetch link previews, open pages |
| `VIBRATE` | ✅ kept — haptics |
| `com.google.android.gms.permission.AD_ID` | ❌ removed (no ads/tracking) |
| `SYSTEM_ALERT_WINDOW` | ❌ removed (no overlays) |
| `READ_EXTERNAL_STORAGE` / `WRITE_EXTERNAL_STORAGE` | ❌ removed (scoped storage + SAF only) |

Re-verify anytime with:
```bash
npx expo prebuild -p android --clean
grep -n "uses-permission" android/app/src/main/AndroidManifest.xml
```
